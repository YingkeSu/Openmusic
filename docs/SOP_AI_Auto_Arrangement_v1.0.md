# Piano for AI SOP（V1.0）

- 版本：V1.0
- 日期：2026-03-01
- 适用范围：桌面本地优先的 AI 自动编曲系统（MVP）
- 关键约束：音频与视频必须本地渲染

## 1. 目标

规范从“用户输入意图”到“导出 MusicXML + MIDI + MP4”的标准流程，确保结果可复现、可回滚、可验收。

## 2. 角色与职责

1. 产品：维护需求、验收标准、风格优先级。
2. 算法/后端：LLM 编曲、乐谱编译、本地音频与视频渲染管线。
3. 客户端：桌面端交互、谱面拖拽编辑、导出与任务状态管理。
4. 测试：功能验证、性能验证、导出一致性验证。

## 3. 标准输入输出

1. 输入字段：`style`、`mood`、`tempo`、`key`、`duration_sec`、`difficulty`、`reference`。
2. 时长限制：`duration_sec <= 60`。
3. 必须导出：`*.musicxml`、`*.mid`、`*.mp4`。
4. 推荐附加导出：`*.wav`（便于质量复核）。

## 4. 端到端操作 SOP

### 4.1 新建项目

1. 用户在桌面端创建曲目项目，填写意图参数。
2. 系统校验字段完整性与时长上限（60 秒）。
3. 生成项目目录：`projects/<project_id>/`。

### 4.2 AI 编曲（结构化）

1. LLM 输出必须为结构化编曲 JSON（不得直接作为最终谱面）。
2. 执行规则校验：音高范围、节拍一致性、小节时值闭合。
3. 校验失败则自动重试，最多 3 次；仍失败则进入人工调整模式。

### 4.3 乐谱编译

1. 使用本地编译器将结构化 JSON 转为 `MusicXML` 与 `MIDI`。
2. 生成简谱文本用于快速预览。
3. 记录版本：`v001` 起，后续编辑递增。

### 4.4 本地音频渲染

1. 输入：`MIDI` + 本地钢琴 `SoundFont`。
2. 本地合成为 `WAV`。
3. 若失败，检查 SoundFont 路径、采样率与 MIDI 完整性后重试。

### 4.5 本地视频渲染

1. 从 `MusicXML` 生成谱面可视化（SVG/Canvas 帧）。
2. 基于时间映射对音符上色：
   - 已播放：黑色
   - 未播放：淡灰
3. 合成帧序列与 WAV 为 `MP4`。

### 4.6 手动微调（优先拖拽）

1. 用户可直接拖拽音符进行编辑。
2. 编辑行为必须回写结构化数据模型，而非只改视觉层。
3. 保存后触发增量重渲染（先音频，后视频）。

### 4.7 导出

1. 一键导出 `MusicXML + MIDI + MP4`。
2. 导出前执行一致性检查：当前编辑版本号与渲染产物版本号一致。
3. 导出后生成 `manifest.json` 记录文件哈希与时间戳。

## 5. 风格处理 SOP（古风优先）

1. MVP 预置 `ancient_cn` 风格模板（古风）。
2. 用户可改为任意风格；系统仍走同一渲染架构。
3. 风格仅影响以下层：
   - LLM 提示词与约束
   - 和声/旋律规则模板
   - 质量评估规则
4. 风格不影响以下层：
   - MusicXML/MIDI 编译
   - 本地音频合成
   - 本地视频编码

## 6. 质量门禁（DoD）

1. 功能通过：可从文本意图完整生成并导出三类文件。
2. 音画同步误差：小于等于 80ms。
3. 单曲 60 秒渲染耗时：在目标设备可接受（需记录机器配置）。
4. 拖拽编辑可用：编辑后重渲染结果可回听、可回放、可导出。

## 7. 异常处理 SOP

1. LLM 输出异常：自动重试并降级到模板化生成。
2. 本地渲染失败：记录 stderr、保留中间产物、提示可重试步骤。
3. 导出失败：回滚导出状态，不回滚项目编辑数据。

## 8. 版本与回滚

1. 每次“保存编辑”都创建新版本快照。
2. 支持回滚到任一历史版本并重新导出。
3. 回滚不删除历史，仅新增“回滚版本”。
