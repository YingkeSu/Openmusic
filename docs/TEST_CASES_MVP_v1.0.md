# 测试用例（MVP Test Cases V1.0）

## 1. 功能用例

### TC-F-001 文本意图生成成功

1. 前置：服务可用。
2. 步骤：提交合法意图（60秒、古风、88BPM）。
3. 期望：产出 `score.json + MusicXML + MIDI`。

### TC-F-002 时长超限拦截

1. 步骤：提交 `duration_sec=61`。
2. 期望：返回参数错误，提示超限。

### TC-F-003 本地音频渲染成功

1. 输入：有效 MIDI + piano.sf2。
2. 期望：生成 WAV，可正常播放。

### TC-F-004 本地视频渲染成功

1. 输入：MusicXML + WAV。
2. 期望：生成 MP4，进度高亮规则正确。

### TC-F-005 导出三件套成功

1. 步骤：执行导出。
2. 期望：导出目录同时存在 MusicXML、MIDI、MP4。

## 2. 编辑用例

### TC-E-001 拖拽升高音高

1. 步骤：拖拽选中音符上移。
2. 期望：`score.json` 对应音符 pitch 更新。

### TC-E-002 编辑后重渲染

1. 步骤：保存拖拽编辑并重渲染。
2. 期望：生成新版本产物，版本号递增。

### TC-E-003 连续编辑回滚

1. 步骤：连续 3 次编辑后回滚到 v001。
2. 期望：v001 产物可恢复并导出。

## 3. 稳定性用例

### TC-S-001 渲染失败重试

1. 前置：故意设置错误 SoundFont 路径。
2. 期望：失败后可重试，错误日志可追踪。

### TC-S-002 批量任务串行执行

1. 步骤：连续提交 5 个生成任务。
2. 期望：任务状态正确，失败不影响其他任务。

## 4. 同步性与性能用例

### TC-P-001 音画同步偏差

1. 步骤：检测 MP4 关键拍点与音频峰值对齐。
2. 期望：偏差 <= 80ms。

### TC-P-002 60 秒曲目渲染时长

1. 步骤：记录完整链路耗时。
2. 期望：满足目标设备阈值（由基准计划定义）。
