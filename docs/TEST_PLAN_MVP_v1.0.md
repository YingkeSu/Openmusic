# 测试计划（MVP Test Plan V1.0）

## 1. 计划信息

- 版本：V1.0
- 对应需求：PRD V1.0 + SPEC V1.0
- 测试对象：桌面本地自动编曲 MVP

## 2. 入口条件

1. 核心接口可调用。
2. 本地依赖安装完成（music21、FluidSynth、FFmpeg、谱面渲染引擎）。
3. 提供最小可用 SoundFont 与样例输入。

## 3. 退出条件

1. P0/P1 缺陷清零。
2. P2 缺陷有明确处置计划。
3. 所有阻断用例通过。
4. 导出三件套通过验收。

## 4. 测试阶段

1. 阶段 A：功能冒烟。
2. 阶段 B：核心链路集成。
3. 阶段 C：端到端回归。
4. 阶段 D：性能与稳定性。

## 5. 关键场景

1. 生成 60 秒古风曲并导出三件套。
2. 切换为自定义风格并成功渲染。
3. 拖拽修改关键音符后重渲染成功。
4. 渲染失败可重试并记录日志。

## 6. 缺陷管理

1. 严重级别：P0/P1/P2/P3。
2. 缺陷字段：`模块、版本、复现步骤、期望、实际、日志、截图`。
3. 回归规则：缺陷修复后必须补对应回归用例。

## 7. 交付产物

1. 测试执行报告。
2. 缺陷统计报告。
3. 发布建议结论（Go/No-Go）。
