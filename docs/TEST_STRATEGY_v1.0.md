# 测试策略（Test Strategy V1.0）

## 1. 测试目标

1. 验证从生成到导出的端到端正确性。
2. 验证本地渲染链路稳定性。
3. 验证拖拽编辑后的重渲染一致性。

## 2. 测试范围

1. 功能测试：生成、编译、音频渲染、视频渲染、导出。
2. 集成测试：UI -> Orchestrator -> Renderers。
3. 回归测试：风格参数、编辑流程、导出流程。
4. 非功能测试：性能、稳定性、错误恢复。

## 3. 不在本期范围

1. 云端渲染链路测试。
2. 多乐器编排复杂测试。
3. 跨平台 UI 细节一致性测试（先聚焦主目标平台）。

## 4. 测试分层

1. 单元测试：
   - 编曲 JSON 校验器
   - 乐谱规则检查器
   - 编辑变更应用器
2. 集成测试：
   - JSON -> MusicXML/MIDI
   - MIDI -> WAV
   - MusicXML + WAV -> MP4
3. 端到端测试：
   - 文本意图 -> 三件套导出
   - 拖拽编辑 -> 重渲染 -> 导出

## 5. 质量阈值

1. 有效输入下 E2E 成功率 >= 95%。
2. 音画同步偏差 <= 80ms。
3. 60 秒曲目全链路渲染达到目标设备可接受时延。

## 6. 风险驱动优先级

1. P0：导出失败、渲染失败、音画严重不同步。
2. P1：拖拽编辑回写错误、版本错乱。
3. P2：风格一致性偏差、UI 可用性细节。
